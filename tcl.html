Transaction Control Language (TCL)

Question 1: Grant SELECT privilege on EMP table to a user named USER1.

(GRANT – BASIC SELECT)

SQL Command:
GRANT SELECT ON EMP TO USER1;
Output:
Grant succeeded.
Question 2: Grant SELECT and INSERT privileges on EMP table to USER2.

(GRANT – MULTIPLE PRIVILEGES)

SQL Command:
GRANT SELECT, INSERT ON EMP TO USER2;
Output:
Grant succeeded.
Question 3: Grant all privileges on DEPT table to USER3.

(GRANT – ALL PRIVILEGES)

SQL Command:

Output:
Grant succeeded.
Question 4: Grant SELECT privilege on EMP table to USER4 with GRANT OPTION.

(GRANT – WITH GRANT OPTION)

Grant succeeded.

Question 19: Insert a new employee record into EMP table and commit the changes.

(COMMIT – INSERT)

SQL Command:
INSERT INTO EMP (EMPNO, ENAME, JOB, SAL, DEPTNO) VALUES (9001, 'JOHN', 'ANALYST', 3500, 20); COMMIT;
Output:
1 row inserted. Commit complete.
Question 20: Update salary of employees in department 10 and commit the changes.

Question 24: Insert a new employee into EMP table but rollback the transaction.

Question 27: Perform multiple operations (INSERT, UPDATE, DELETE) and rollback all changes.

(ROLLBACK – MULTIPLE OPERATIONS)

SQL Command:
INSERT INTO EMP (EMPNO, ENAME, SAL, DEPTNO) VALUES (9003, 'SARA', 3500, 10); UPDATE EMP SET SAL = SAL + 100 WHERE EMPNO = 9003; DELETE FROM EMP WHERE SAL < 1500; ROLLBACK;
Output:
1 row inserted. 1 row updated. 2 rows deleted. Rollback complete. All operations cancelled.
Question 28: Update department location and rollback before committing.

(ROLLBACK – DEPT TABLE)

SQL Command:
UPDATE DEPT SET LOC = 'FLORIDA' WHERE DEPTNO = 30; -- Verify before commit SELECT * FROM DEPT WHERE DEPTNO = 30; ROLLBACK;
Output:
1 row updated. DEPTNO=30, DNAME=SALES, LOC=FLORIDA Rollback complete. Update cancelled and LOC reverted.
Question 29: Insert a new employee, update salary, and commit only after verifying results.

(TRANSACTION – COMMIT CONTROL)

SQL Command:
INSERT INTO EMP (EMPNO, ENAME, JOB, SAL, DEPTNO) VALUES (9004, 'EMMA', 'ANALYST', 3200, 20); UPDATE EMP SET SAL = 3500 WHERE EMPNO = 9004; -- Verify SELECT * FROM EMP WHERE EMPNO = 9004; COMMIT;
Output:
1 row inserted. 1 row updated. EMPNO=9004, ENAME=EMMA, JOB=ANALYST, SAL=3500, DEPTNO=20 Commit complete.
Question 30: Insert multiple employees, rollback one operation using SAVEPOINT, then commit remaining.

(TRANSACTION – SAVEPOINT + COMMIT)

SQL Command:
INSERT INTO EMP (EMPNO, ENAME, SAL, DEPTNO) VALUES (9005, 'DAVID', 2800, 30); SAVEPOINT sp1; INSERT INTO EMP (EMPNO, ENAME, SAL, DEPTNO) VALUES (9006, 'LISA', 3900, 20); ROLLBACK TO sp1; -- First insert kept, second insert rolled back COMMIT;
Output:
1 row inserted. Savepoint sp1 created. 1 row inserted. Rollback to savepoint sp1 complete. LISA insert cancelled. Commit complete. DAVID remains.
Question 31: Create a transaction where some operations are committed and others are rolled back.

(TRANSACTION – PARTIAL CONTROL)

SQL Command:
INSERT INTO DEPT VALUES (60, 'TRAINING', 'ATLANTA'); COMMIT; UPDATE DEPT SET LOC = 'HOUSTON' WHERE DEPTNO = 60; ROLLBACK;
Output:
1 row inserted. Commit complete. TRAINING dept added. 1 row updated. Rollback complete. Location update cancelled.
Question 32: Demonstrate use of SAVEPOINT in EMP table operations.

(TRANSACTION – SAVEPOINT)

SQL Command:
INSERT INTO EMP (EMPNO, ENAME, SAL, DEPTNO) VALUES (9007, 'MARK', 2600, 10); SAVEPOINT sp_insert; UPDATE EMP SET SAL = SAL + 200 WHERE EMPNO = 9007; SAVEPOINT sp_update; DELETE FROM EMP WHERE EMPNO = 9007; COMMIT;
Output:
1 row inserted. Savepoint sp_insert created. 1 row updated. Savepoint sp_update created. 1 row deleted. Commit complete. All operations saved.
Question 33: Perform update on EMP table and rollback to a specific SAVEPOINT.

(TRANSACTION – ROLLBACK TO SAVEPOINT)

SQL Command:
UPDATE EMP SET SAL = 3000 WHERE JOB = 'CLERK'; SAVEPOINT sp_clerk_update; UPDATE EMP SET SAL = 4500 WHERE JOB = 'MANAGER'; ROLLBACK TO sp_clerk_update; -- Manager update rolled back, clerk update remains COMMIT;
Output:
1 row updated. Savepoint sp_clerk_update created. 3 rows updated. Rollback to savepoint complete. Manager update cancelled. Commit complete. Clerk salaries changed to 3000.
Question 34: Grant privileges to a user, perform DML operations, and commit changes.

Output:
Grant succeeded. 1 row inserted. Commit complete.
Question 35: Revoke privileges and verify access restrictions using EMP table.

(REVOKE – PRACTICAL SCENARIO)

SQL Command:
REVOKE INSERT, UPDATE ON EMP FROM USER10; -- User10 can only SELECT now -- Attempt to INSERT (will fail) INSERT INTO EMP (EMPNO, ENAME, SAL, DEPTNO) VALUES (9009, 'CAROL', 3400, 20);
Output:
Revoke succeeded. Error: ORA-01031: insufficient privileges (INSERT privilege no longer available to USER10)
